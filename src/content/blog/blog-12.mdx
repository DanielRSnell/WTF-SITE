## WooCommerce Database Optimization for Global Stores

### Regional Database Architecture

For global WooCommerce stores, implement a multi-region database strategy:

```php
// Database configuration for regional optimization
class WooThatsFast_Global_DB {
    private $regional_connections = [];
    
    public function __construct() {
        // Primary database (read/write)
        $this->regional_connections['primary'] = new mysqli(
            DB_PRIMARY_HOST,
            DB_USER, 
            DB_PASSWORD,
            DB_NAME
        );
        
        // Regional read replicas
        $this->regional_connections['eu-replica'] = new mysqli(
            getenv('DB_EU_HOST'),
            DB_USER,
            DB_PASSWORD, 
            DB_NAME
        );
        
        $this->regional_connections['asia-replica'] = new mysqli(
            getenv('DB_ASIA_HOST'),
            DB_USER,
            DB_PASSWORD,
            DB_NAME
        );
    }
    
    public function get_regional_connection($region) {
        $customer_region = $this->detect_customer_region();
        
        // Route to appropriate database based on operation
        if (in_array($_SERVER['REQUEST_METHOD'], ['POST', 'PUT', 'DELETE'])) {
            // Write operations go to primary
            return $this->regional_connections['primary'];
        }
        
        // Read operations use regional replica if available
        $replica_key = $customer_region . '-replica';
        if (isset($this->regional_connections[$replica_key])) {
            return $this->regional_connections[$replica_key];
        }
        
        // Fallback to primary
        return $this->regional_connections['primary'];
    }
}
```

```php
// Regional caching configuration
class WooThatsFast_Regional_Cache {
    
    public static function init_regional_caching() {
        $customer_region = self::get_customer_region();
        
        switch($customer_region) {
            case 'europe':
                // GDPR-compliant caching
                self::configure_gdpr_cache();
                break;

        // Set shorter cache times for user-specific data
        add_filter('wc_session_expiry', function($expiry) {

        });
        
        // Disable persistent cart for privacy

        // Use cookie-less caching where possible

            echo '<link rel="preload" href="/wp-content/uploads/mobile-hero.webp" as="image">';
```

## Building the Regional Frontend Experience

### Geo-Located Performance Optimization

When serving WooCommerce stores globally, optimize based on detected location:

#### Server-Side Region Detection

```php
// Detect customer region for optimization
function woothatsfast_detect_region() {
    // Check for override cookie first
    if (isset($_COOKIE['woothatsfast_region'])) {
        return sanitize_text_field($_COOKIE['woothatsfast_region']);
    }
    
    // Use CloudFlare country header if available
    if (isset($_SERVER['HTTP_CF_IPCOUNTRY'])) {
        $country = $_SERVER['HTTP_CF_IPCOUNTRY'];
        return woothatsfast_country_to_region($country);
    }
    
    // Fallback to IP geolocation
    $ip = $_SERVER['REMOTE_ADDR'];
    $region = woothatsfast_geolocate_ip($ip);
    
    return $region ?: 'us'; // Default to US
}

function woothatsfast_country_to_region($country) {
    $region_map = [
        'US' => 'us',
        'CA' => 'us', // Canada uses US optimization
        'GB' => 'europe',
        'FR' => 'europe', 
        'DE' => 'europe',
        'IT' => 'europe',
        'ES' => 'europe',
        'SG' => 'asia_pacific',
        'JP' => 'asia_pacific',
        'AU' => 'asia_pacific',
        'HK' => 'asia_pacific'
    ];
    
    return $region_map[$country] ?? 'us';
}
```

#### JavaScript Regional Optimization

```javascript
// Client-side regional optimization
class WooThatsFastRegional {
    constructor() {
        this.region = this.detectRegion();
        this.initRegionalOptimizations();
    }
    
    detectRegion() {
        // Check for server-set region first
        const serverRegion = document.body.dataset.woothatsfast_region;
        if (serverRegion) return serverRegion;
        
        // Fallback to timezone detection
        const timezone = Intl.DateTimeFormat().resolvedOptions().timeZone;
        
        if (timezone.includes('America')) return 'us';
        if (timezone.includes('Europe')) return 'europe'; 
        if (timezone.includes('Asia') || timezone.includes('Australia')) {
            return 'asia_pacific';
        }
        
        return 'us'; // Default
    }
    
    initRegionalOptimizations() {
        switch(this.region) {
            case 'europe':
                this.initGDPROptimizations();
                break;
            case 'asia_pacific':
                this.initMobileOptimizations();
                break;
            default:
                this.initStandardOptimizations();
        }
    }
    
    initGDPROptimizations() {
        // Minimize data collection
        this.disableNonEssentialTracking();
        
        // Show privacy-focused UI
        document.body.classList.add('gdpr-mode');
        
        // Optimize for privacy-conscious users
        this.enablePrivacyMode();
    }
    
    initMobileOptimizations() {
        // Aggressive mobile optimization
        this.enableDataSaver();
        this.optimizeForSlowConnections();
        
        // Touch-first interface improvements
        document.body.classList.add('mobile-optimized');
    }
    
    enableDataSaver() {
        // Lazy load all non-critical images
        const images = document.querySelectorAll('img:not([data-critical])');
        images.forEach(img => {
            img.loading = 'lazy';
        });
        
        // Reduce image quality for slower connections
        if (navigator.connection && navigator.connection.effectiveType === '2g') {
            this.reduceImageQuality();
        }
    }
}
```

## URL Strategies for Global WooCommerce Stores

WooThatsFast supports all common URL patterns for global stores:

### 1. Subdirectory Approach

Example: `yourstore.com/eu/products`

```apache
# .htaccess regional routing
RewriteEngine On

# Route EU traffic to EU-optimized version
RewriteCond %{HTTP_CF_IPCOUNTRY} ^(GB|FR|DE|IT|ES|NL)$
RewriteCond %{REQUEST_URI} !^/eu/
RewriteRule ^(.*)$ /eu/$1 [R=302,L]

# Route APAC traffic to APAC-optimized version  
RewriteCond %{HTTP_CF_IPCOUNTRY} ^(SG|JP|AU|HK)$
RewriteCond %{REQUEST_URI} !^/apac/
RewriteRule ^(.*)$ /apac/$1 [R=302,L]
```

### 2. Subdomain Approach

Example: `eu.yourstore.com/products`

```php
// Subdomain-based regional optimization
function woothatsfast_subdomain_region() {
    $host = $_SERVER['HTTP_HOST'];
    
    if (strpos($host, 'eu.') === 0) {
        return 'europe';
    } elseif (strpos($host, 'apac.') === 0) {
        return 'asia_pacific';
    }
    
    return 'us'; // Default
}
```

### 3. Country-Specific Domains with Shared Infrastructure

Example: `yourstore.co.uk` vs `yourstore.de`

```php
// Multi-domain performance optimization
function woothatsfast_domain_region() {
    $domain = $_SERVER['HTTP_HOST'];
    
    $domain_regions = [
        'yourstore.co.uk' => 'europe',
        'yourstore.de' => 'europe', 
        'yourstore.com.au' => 'asia_pacific',
        'yourstore.sg' => 'asia_pacific'
    ];
    
    return $domain_regions[$domain] ?? 'us';
}
```
